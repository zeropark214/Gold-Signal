# UI/UX 규칙

## 참고 원칙

Byul과 스톡나우의 공개 앱 설명, 업데이트 내역, 보도자료를 참고해 Gold Signal의 UI/UX 규칙을 정리한다. 목표는 거래 앱이 아니라 실시간 금속 시장 뉴스와 지표를 빠르게 확인하는 모니터링 앱이다.

## 적용 규칙

### 1. 실시간 피드 우선

- 홈과 뉴스 화면은 최신 정보가 먼저 보여야 한다.
- 지표, 뉴스, 커뮤니티는 모두 시간 흐름이 느껴지는 리스트 구조를 기본으로 한다.
- 속보, 중요, 일반 같은 상태 배지를 명확히 표시한다.

### 2. 브리핑 중심

- 뉴스 화면 상단에는 오늘의 시장 브리핑을 제공한다.
- 사용자가 긴 기사 목록을 읽기 전, 핵심 요약과 영향 요인을 먼저 파악할 수 있어야 한다.
- 향후 오디오 브리핑과 자막형 브리핑을 연결할 수 있는 구조로 둔다.

### 3. 영향 점수와 태그

- 뉴스에는 금값 영향 점수를 표시한다.
- 뉴스에는 금리, 달러, 인플레이션, 중앙은행 금 매입 등 영향 요인 태그를 붙인다.
- 관련 자산을 함께 보여 사용자가 어떤 가격을 확인해야 하는지 알 수 있게 한다.

### 4. 다크 모니터링 UI

- 전체 배경은 어두운 톤을 사용해 야간 확인과 장시간 모니터링에 적합하게 만든다.
- 카드와 리스트는 고대비로 구분하되 과도한 장식은 피한다.
- 상승, 하락, 속보, 중요 상태는 색으로 빠르게 구분한다.

### 5. 워치리스트 확장성

- 뉴스의 관심 요인 설정은 향후 개인화 워치리스트로 확장한다.
- 사용자는 금리, 달러, CPI, 중앙은행 금 매입 등 본인이 중요하게 보는 요인을 선택할 수 있어야 한다.

### 6. 번역과 원문 병기

- 한국어 번역 헤드라인을 먼저 보여준다.
- 원문 헤드라인과 출처를 함께 표시해 신뢰도를 보완한다.
- 원문 전체는 재게시하지 않고 원문 링크로 연결한다.

### 7. 접근성

- 다크 UI에서도 본문, 배지, 버튼의 대비가 충분해야 한다.
- 버튼과 리스트 행은 모바일 기준 최소 44px 이상으로 유지한다.
- 아이콘 버튼에는 `aria-label`을 제공한다.

### 8. 커뮤니티 연결

- 속보와 지표에서 관련 커뮤니티 토론으로 자연스럽게 이동할 수 있어야 한다.
- 댓글 수와 반응 수를 보여 사용자가 시장 반응을 빠르게 파악하도록 한다.
